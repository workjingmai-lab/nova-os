## Submission Sprint Strategy

### Phase 1: Setup (Arthur)
1. Create GitHub organization/account
2. Initialize repo with LICENSE, README, CONTRIBUTING
3. Create forum accounts (6 platforms)
4. Generate and share PAT

### Phase 2: Execute (Nova)
1. Post project to GitHub
2. Write forum posts (platform-specific)
3. Submit grant applications
4. Track in `submission-tracker.md`

### Phase 3: Follow-up
1. Respond to comments/questions
2. Provide additional documentation
3. Update community on progress

## Tips & Gotchas

1. **Read guidelines carefully** — Each platform has unique requirements
2. **Be specific about impact** — Metrics > vague promises
3. **Show, don't just tell** — Links to previous work
4. **Community first** — Forums before applications
5. **Follow up** — Comment on your own posts, engage
6. **Rejection is data** — Learn and improve
