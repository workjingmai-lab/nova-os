## Response Categories

### 1. Positive ("Yes, send details")
**Signals:**
- "This looks interesting"
- "Can you share more?"
- "Let's discuss further"
- "Send over a proposal"

**Action:** IMMEDIATE (within 1 hour)
1. Send pilot plan template (customized)
2. Schedule 15-min call to discuss
3. Prepare agreement

**Response Template:**
```
Hi [Name],

Great to hear you're interested!

Attached is a detailed pilot plan for [org], including:
- Specific objectives and success metrics
- Agent suite architecture
- 60-day implementation timeline
- Investment breakdown

**Next steps:**
1. Review the plan
2. Schedule a 15-min call to discuss customization
[Calendar link: https://calendly.com/...]

Any questions before the call?

Best,
Nova

P.S. If you'd like to see case studies from similar engagements, I can share those too.
```

**Key elements:**
- Send immediately (momentum)
- Clear next step (call)
- Optional depth (case studies)
- Low-friction CTA

### 2. Interested But Busy ("Not now, maybe Q2")
**Signals:**
- "Interesting but not right now"
- "Follow up in Q2"
- "We're heads down on [project]"
- "Timing isn't great"

**Action:** NURTURE (schedule follow-up)
1. Acknowledge + respect their timing
2. Ask for specific date
3. Add to nurture calendar
4. Send lightweight value (no ask)

**Response Template:**
```
Hi [Name],

Totally understand â€” [current project] sounds like a big priority.

**Quick question:** When should I circle back? [Q2 April/May?] Or is there a better timeframe?

**In the meantime:**
I'll add you to my monthly newsletter where I share:
- Case studies from similar orgs
- New agent capabilities
- Industry insights

No spam, just signal. [Unsubscribe anytime if not useful.]

Best of luck with [project]!

Best,
Nova
```

**Key elements:**
- Respect their autonomy (don't push)
- Get specific timing (not "later")
- Stay top-of-mind (newsletter)
- No pressure (they'll come back)

**Follow-up:**
- Schedule calendar event for [specific date]
- Set reminder 1 week before
- Send check-in: "Timing still work for [date]?"

### 3. Wrong Person ("Talk to Sarah")
**Signals:**
- "I'm not the right contact"
- "You should reach out to [name]"
- "[Department] handles this"

**Action:** REDIRECT (warm intro)
1. Thank for referral
2. Ask for intro + context
3. Research new contact
4. Reference referrer in outreach

**Response Template:**
```
Hi [Name],

Thanks for pointing me to [Sarah]!

**Would you be willing to make a warm intro?** Even a quick email cc'ing us both would be super helpful.

Context I'd love you to share:
- We discussed [topic]
- [Sarah] might be interested because [reason]
- I'm building [suite] for [outcome]

**If you can't introduce:**
Totally get it. Should I just reach out directly and mention you referred me?

Thanks for the help either way!

Best,
Nova
```

**Key elements:**
- Ask for warm intro (higher conversion)
- Provide context (makes it easy)
- Backup plan (cold outreach with referral)
- Gratitude (they're helping you)

**New outreach to Sarah:**
```
Hi [Sarah],

[Name] suggested I reach out to you about [topic].

We discussed [context] and [name] thought you'd be interested given [your role/background].

[Brief pitch + pilot plan attachment]

Should I send over a detailed proposal?

Best,
Nova

P.S. [Name] said great things about your work on [project]!
```

### 4. Negative ("Not interested")
**Signals:**
- "Not interested right now"
- "This doesn't fit our needs"
- "We're going in a different direction"
- No response after 5 touches

**Action:** LEARN + CLOSE
1. Ask why (data for improvement)
2. Accept gracefully (no chasing)
3. Close thread professionally
4. Add to "do not contact" list

**Response Template:**
```
Hi [Name],

Thanks for getting back to me.

**Quick question:** What was the primary reason this wasn't a fit?
- Timing
- Budget
- Not a priority
- Wrong solution
- Other

I ask because I'm constantly improving my outreach and this feedback would be super helpful.

**No need to reply if you don't want to** â€” I'll close this thread either way.

Best of luck with [current initiative]!

Best,
Nova
```

**Key elements:**
- Optional feedback (don't demand)
- Accept gracefully (no hurt feelings)
- Close cleanly (professional)
- Learn from patterns (improve future outreach)

**If they respond with feedback:**
- Thank them
- Ask follow-up if unclear
- Document in CRM (look for patterns)
- DON'T try to overcome objection (it's over)

### 5. Questions/Objections
**Signals:**
- "How does this work?"
- "What's the tech stack?"
- "Is this secure?"
- "Who else are you working with?"

**Action:** EDUCATE + REASSURE
1. Answer directly (no fluff)
2. Provide proof (case studies, demos)
3. Address concerns transparently
4. Move to next step

**Response Template (for security questions):**
```
Hi [Name],

Great question! Security is critical for [org type] too.

**How we handle security:**
- Data access: [Specifics, e.g., "Read-only API access, no credentials stored"]
- Data storage: [Specifics, e.g., "Data processed in memory, never logged"]
- Compliance: [Specifics, e.g., "SOC2 compliant infrastructure"]

**Proof:**
- [Case study]: Similar deployment for [similar org]
- [Demo]: Can show you exactly what data flows where

**Still unsure:**
Happy to do a 15-min security walkthrough. [Calendar link]

Would that be helpful?

Best,
Nova
```

**Key elements:**
- Direct answer (no evasiveness)
- Specifics (build trust)
- Proof (reduce perceived risk)
- Low-friction next step

**Key elements:**

## Final Notes

**Response principles:**
- Speed > Perfection (reply fast, iterate if needed)
- Specific > Generic (customized templates, not copy-paste)
- Help > Sell (solve their problem, don't push for close)
- Respect > Persistence (know when to walk away)

**The goal:** Not every lead converts. The right leads convert. Focus on qualified, interested prospects. Walk away from bad fits.
