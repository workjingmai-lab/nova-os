# lead-prioritizer.py

Rank revenue pipeline leads by ROI potential to focus on highest-value opportunities first.

## What It Does

Calculates priority scores (0-100) for all pipeline leads based on:
- **Potential value** (0-60 points) â€” Higher value = higher score
- **Readiness bonus** (+30) â€” No blockers = ready NOW
- **Blocker penalty** (-40) â€” Blocked = needs Arthur action
- **Category bonus** (+5-10) â€” Grants (higher conversion) > services

## Why It Matters

Prevents wasting time on low-value or blocked leads. Forces focus on highest ROI opportunities first. Priority tiers:
- **HIGH (70+):** Execute immediately (typically $30K-$50K each)
- **MEDIUM (40-69):** Execute after HIGH
- **LOW (<40):** Execute later or skip

## Usage

```bash
# All leads, ranked by priority
python3 tools/lead-prioritizer.py

# Top 10 leads only
python3 tools/lead-prioritizer.py --top 10

# Services only
python3 tools/lead-prioritizer.py --category service

# Ready-to-send only (zero blockers)
python3 tools/lead-prioritizer.py --ready

# Blocked items only (needs unblocking)
python3 tools/lead-prioritizer.py --blocked

# Show scoring reasons (debug)
python3 tools/lead-prioritizer.py --reasons
```

## Score Formula

```
Score = Value + Readiness + Category

Value (0-60):     min(potential, 60)  â€” $60K+ = max points
Readiness (+30):  No blockers
Readiness (-40):  Blocked
Category (+10):   Grants (higher conversion)
Category (+5):    Services (direct outreach)
```

- EF $40K, ready: 60 + 30 + 5 = 95 â†’ HIGH âœ…
