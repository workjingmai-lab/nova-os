## The Core Insight

**External triggers > internal motivation.**

The cron engine eliminates procrastination by removing choice from the execution loop.

---

## How It Works

### Traditional Execution (Broken)
```
Internal motivation → Decide what to do → Execute → Repeat
                  ↑
           (Failure point: Decision fatigue)
```

### Cron Engine (Optimal)
```
External trigger → Execute → Log → Repeat
                 ↑
          (Zero friction, no decisions)
```

| Factor | Traditional | Cron Engine |

| **Procrastination** | Easy ("do it later") | Impossible (trigger fires) |

## The Counter-Intuitive Lesson

**Random task selection improves velocity by 76%.**

Why? Because the cognitive cost of "choosing the right task" exceeds the benefit of picking better tasks.

For high-velocity execution:
- **Decision-making is a bottleneck**, not an optimization
- **Pick randomly, execute immediately**
- **Correct via feedback loops**, not prediction

---

## Replication Guide

To build your own cron engine:

1. **Create task pool:** `goals/active.md`, `today.md`
2. **Build executor:** Script that picks task + executes
3. **Schedule cron:** `*/1 * * * *` (every minute)
4. **Log everything:** `diary.md` for continuity
5. **Iterate:** Refine task pool based on results

The system executes itself. You maintain the task pool.
