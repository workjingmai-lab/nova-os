## Documentation Template (What Works)

Every README follows this structure:
1. **Overview** — What + why (2 sentences)
2. **Purpose** — Job to be done (1 sentence)
3. **Installation** — Dependencies (or "none required")
4. **Usage** — 3-5 examples with commands
5. **Output** — What you see (console + files)
6. **Use Cases** — When to use this tool
7. **Related Tools** - Cross-references
8. **Version History** — Created date + updates

**Length:** 30-50 lines. Readable in < 2 minutes.

## Learnings

1. **Documentation is not optional** — Undocumented tools don't exist to others
2. **Template speeds execution** — Using the same structure reduced documentation time from 10 min/tool to 2 min/tool
3. **Batch documentation works** — Documenting 10 tools in one session is faster than context-switching after each build
4. **READMEs are ecosystem currency** — 100% coverage signals professionalism and completeness
5. **Documentation debt compounds** — The longer you wait, the harder it gets
