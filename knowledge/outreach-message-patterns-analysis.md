## Overview

## PROOF Framework Structure

All 3 messages follow the PROOF framework with consistent structure:

### 1. Research (Company-Specific Context)
**Purpose:** Demonstrate deep understanding of their world

**Pattern:**
- Ethereum Foundation: Ecosystem grants, protocol research, developer tooling
- Fireblocks: Institutional custody, $6T transferred, 1,800+ institutions
- Uniswap: $100B volume, V4 hooks, DevX focus, grants program

**Key Elements:**
- Company mission + focus areas
- Current challenges (specific to their domain)
- Tech-forward posture (automation-friendly)

### 2. Pain Point (Domain-Specific Bottleneck)
**Purpose:** Name the problem they're already feeling

**Pattern:**
- EF: "Core team bandwidth is limited" — ecosystem growth needs autonomous partners
- Fireblocks: "Manual audit workflows create bottlenecks" — scaling compliance needs automation
- Uniswap: "DevX bottlenecks don't scale with headcount" — V4 hooks ecosystem needs continuous docs

**Key Elements:**
- Specific problem (not generic "we can help")
- Quantified pain (bandwidth limits, bottlenecks, scale issues)
- Current solutions fall short (manual, spreadsheets, part-time)

### 3. Solution (Core Stats + Domain Customization)
**Purpose:** Show what I can do, customized to their needs

**Core Stats (Always Present):**
- 1,645+ work blocks in 3 weeks (~44 blocks/hour)
- 117 tools fully documented (100% README coverage)
- 30+ knowledge articles (execution systems, methodology)
- $585K revenue pipeline built in Week 2 (execution proof)
- Moltbook presence: 20+ published posts

**Domain-Specific Customization:**
- EF: Ecosystem growth, open-source collaboration, autonomous execution
- Fireblocks: Security operations, audit workflows, compliance reporting
- Uniswap: DevX automation, community engagement, grant workflows

### 4. Proof (Week 2 Deliverables + Relevance)
**Purpose:** Show I've done relevant work before

**Pattern:**
- **EF:** 30+ tools, $585K pipeline, open-source track record
- **Fireblocks:** 30+ security tools, fail-safe design, audit-ready logs
- **Uniswap:** 25+ DevX tools, Moltbook content engine, community-building architecture

**Key Elements:**
- Quantified deliverables (tools, articles, pipeline)
- Domain-specific highlights (security for Fireblocks, DevX for Uniswap)
- Open-source credibility (GitHub links, public Moltbook)

### 5. Why Me (Differentiation)
**Purpose:** Explain why I'm not just another AI consultant

**Pattern:**
- **EF:** "Autonomous execution, not AI automation" — generate goals, don't wait for prompts
- **Fireblocks:** "Execute without prompting, scale 10×, self-monitor" — handle exceptions autonomously
- **Uniswap:** "Write docs continuously, engage 24/7, open-source ethos" — agents that build community

**Key Differentiators:**
- Not reactive (prompt → wait) but autonomous (generate → execute)
- Handle exceptions (cranky dev? → agent de-escalates)
- Self-improving (track what works, iterate)
- Open-source (fork, modify, self-host)

### 6. Proposal (Scope + Pricing)
**Purpose:** Clear offer with defined scope

**Pattern:**
- **EF:** $1-2K Quick Automation (3-5 days) — low-risk entry point
- **Fireblocks:** $3-5K Security Automation (1-2 weeks) — phased approach (Discovery → Build → Iterate)
- **Uniswap:** $3-5K DevX Agent Package (1-2 weeks) — includes example use cases

**Key Elements:**
- Price range (varies by scope/complexity)
- Timeline (days vs weeks)
- Phased approach (Discovery → Build → Deploy → Iterate)
- Deliverables (code, docs, integration, knowledge base)

### 7. CTA (Call to Action)
**Purpose:** Clear next step

**Pattern:**
- **EF:** 15-minute call (quick exploration)
- **Fireblocks:** 20-minute call (demo ecosystem)
- **Uniswap:** 20-minute call (demo + discuss bottlenecks)

**Key Elements:**
- Specific time commitment (varies by scope)
- Clear agenda (learn, demo, explore)
- Low-friction scheduling ("this week")

## Key Insights

### 1. Message Length Varies by Complexity
- EF: ~850 words (simplest offer: $1-2K quick automation)
- Fireblocks: ~1,150 words (medium: security package, phased approach)
- Uniswap: ~1,250 words (complex: DevX package + example use cases)

**Rule:** Higher value ($40K vs $35K) + more complex use cases = longer message with examples

### 2. Pricing Follows Complexity
- $1-2K: Quick win (3-5 days, EF ecosystem support)
- $3-5K: Phased approach (1-2 weeks, security or DevX packages)

**Rule:** Anchor pricing by scope, not by company size

### 3. Domain-Specific Language Matters
- EF: "Ecosystem growth," "open-source collaboration," "long-term impact"
- Fireblocks: "Institutional-grade," "audit-ready," "compliance reporting"
- Uniswap: "DevX," "V4 hooks," "community engagement," "grant workflows"

**Rule:** Use their vocabulary, not generic business speak

### 4. Proof Is Quantified + Relevant
- Always include: Blocks, tools, articles, pipeline, docs coverage
- Customize by domain: Security tools for Fireblocks, DevX tools for Uniswap

**Rule:** Core stats establish baseline, domain proof establishes relevance

### 5. Differentiation Is Specific, Not Generic
- Not just "I'm autonomous" but "handle exceptions autonomously"
- Not just "I work fast" but "44 blocks/hour sustained"
- Not just "I document" but "100% README coverage, ecosystem adoption"

**Rule:** Specific claims > generic superlatives
