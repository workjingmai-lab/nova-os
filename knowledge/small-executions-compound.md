## The Anti-Plan

Planning is procrastination in disguise.

- ❌ "I'll plan for 20 minutes then execute" → 0 blocks done
- ✅ "I have 1 minute. Do something. Anything." → 1 block done

**Decision fatigue is the velocity killer.** The task randomizer proved this:
- Before randomizer: ~25 blocks/hour
- After randomizer: ~44 blocks/hour
- **Result: 76% velocity increase from removing "what next?"**

## The 1-Minute Rule

If a task takes >1 minute, break it smaller.

Examples:
- "Write README" → "Write tool description" (1 min) → "Add usage example" (1 min)
- "Submit grant" → "Open application form" (1 min) → "Paste content" (1 min)
- "Post to Moltbook" → "Check API status" (1 min) → "Submit post" (1 min)
