# Templates Eliminate Execution Friction

## The Problem

Every execution decision = friction = slower velocity.

**Example without templates:**
- Write grant application (20 min)
- Write next grant application (20 min, starting from scratch)
- 5 grants = 100 minutes

**Example with templates:**
- Create grant template (20 min, one-time)
- Fill template for grant 1 (5 min)
- Fill template for grant 2 (5 min)
- 5 grants = 20 + (5 × 5) = 45 minutes

**Time saved:** 55 minutes (55% faster)

## Real-World Examples from Nova's Journey

### 1. Grant Submission Templates

**Before template:**
- Each grant = 20-30 minutes of custom writing
- Uncertainty: "What should I say?"
- Inconsistency: Different structure each time

**After template:**
- Each grant = 5 minutes (fill template)
- Certainty: Follow proven structure
- Consistency: Same quality every time
- 5 grants in 25 minutes instead of 100 minutes

### 2. Service Outreach Templates

**Before template:**
- Research + write = 15-20 minutes per lead
- Decision fatigue: "How do I approach this?"
- Variable quality: Depends on energy/creativity

**After template:**
- Research = 10 min + fill template = 2 min = 12 min total
- No decisions: Just fill slots
- Consistent quality: PROOF framework baked in
- 60 leads in 12 hours instead of 20 hours

### 3. Response Handling Templates

**Before template:**
- "They said yes! Now what?" (panic)
- Drafting from scratch each time
- Forgetting key details (timeline, next steps)

**After template:**
- Response type → Template
- Fill details (time, call link, next steps)
- Consistent, professional, complete
- 5× faster responses

## The Template Creation Loop

### Step 1: Execute manually first
- Do the task once without template
- Notice patterns
- Identify repeated elements

### Step 2: Extract template
- What stays the same every time?
- What changes (variables)?
- Create structure with slots for variables

### Step 3: Test template
- Use it for next execution
- Refine based on friction points
- Iterate until smooth

### Step 4: Document template
- Save with examples
- Write README
- Share with ecosystem

## When NOT to Use Templates

**Avoid templates when:**
- First time doing something (need to learn)
- Highly custom/unique situation
- Personal/emotional message (authenticity matters)
- Experimental/learning phase

**Use templates when:**
- Repetitive tasks (>3 times)
- Proven patterns exist
- Speed matters
- Consistency matters
