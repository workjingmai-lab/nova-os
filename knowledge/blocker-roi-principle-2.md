## The Principle

**Identify blockers by ROI/min. Fix the highest ROI blocker first.**

Simple, powerful, counterintuitive.

---

## Why It Matters

Traditional prioritization:
- "What's most important?" ‚Üí Too vague
- "What's urgent?" ‚Üí Everything feels urgent
- "What do I feel like doing?" ‚Üí Decision fatigue

Blocker ROI prioritization:
- "What unblocks the most value per minute?" ‚Üí Mathematical, clear

---

## The Formula

```
Blocker ROI = Value Unblocked / Time to Fix
```

**Example:**
- Browser restart: 1 min ‚Üí unblocks $180K
- GitHub auth: 8 min ‚Üí unblocks $130K
- Review docs: 30 min ‚Üí unblocks $10K

**Priority order:** Browser ($180K/min) > GitHub ($16K/min) > Review ($0.3K/min)

## How to Apply

### Step 1: List Your Blockers
What's frozen? What can't move forward?

```
- Browser automation blocked ‚Üí Can't submit grants/audit
- GitHub auth missing ‚Üí Can't push repo
- API keys needed ‚Üí Can't access service
- Template missing ‚Üí Can't send proposals
```

### Step 2: Calculate Value Unblocked
What happens when this blocker is gone?

```
- Browser ‚Üí $180K pipeline activated
- GitHub ‚Üí $130K grants submittable
- Templates ‚Üí $122K outreach possible
```

### Step 3: Estimate Fix Time
How long to fix this? (Be honest)

```
- Browser restart ‚Üí 1 minute (one command)
- GitHub auth ‚Üí 8 minutes (login process)
- Create templates ‚Üí 60 minutes (design + write)
```

### Step 4: Calculate ROI
```
Browser: $180K / 1 min = $180K/min
GitHub: $130K / 8 min = $16,250/min
Templates: $122K / 60 min = $2,033/min
```

### Step 5: Execute in Order
Fix highest ROI first. **Don't multitask.**

```
1. Restart browser (1 min) ‚Üí $180K unblocked
2. GitHub auth (8 min) ‚Üí $130K unblocked
3. Templates (60 min) ‚Üí $122K unblocked
```

Total time: 69 minutes
Total value: $432K unblocked
Velocity: **$6.26K/minute**
