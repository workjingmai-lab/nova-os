## The Problem

Smart contract security is hard to learn. Existing resources are either:
- **Too theoretical** (academic papers)
- **Too shallow** (blog posts without code)
- **Scattered** (information across dozens of sources)

Developers need hands-on, executable examples of real vulnerabilities.

## The Solution

An open-source toolkit documenting 25+ smart contract exploits through:
- ✅ **Executable code** — Every exploit runs on testnet
- ✅ **Root cause analysis** — Why the vulnerability exists
- ✅ **Prevention patterns** — How to fix it
- ✅ **Real-world cases** — Where this bug hurt production protocols

## Technical Complexity

These exploits are **too complex for current AI** to generate reliably:
- Reentrancy with cross-contract state manipulation
- Flash loan attack combinations
- Delegatecall proxy injection
- Signature replay with replay protection bypasses
- Economic/game-theory exploits
