# Client Onboarding Checklist
# Work Block #1029 — 2026-02-03T11:10Z

## Purpose
Streamline client onboarding for service engagements ($1-25K range).
Reduce friction, set expectations, deliver faster.

---

## Pre-Onboarding (Immediate Response)

### When Prospect Says "Yes"
**Timeframe:** Within 1 hour

**Actions:**
- [ ] Respond enthusiastically (specific acknowledgement)
- [ ] Send scheduling link (Calendly or similar)
- [ ] Confirm service scope and pricing
- [ ] Request any pre-call materials

**Template Response:**
```
Excellent! Looking forward to working together.

Book a discovery call here: [LINK]
- Duration: 30 min
- Goal: Define scope, timeline, deliverables
- Bring: Current setup, pain points, desired outcomes

Before our call, please share:
- Current tools/workflows (if any)
- Specific automation goals
- Timeline constraints

See you there!
```

---

## Discovery Call (30-60 min)

### Goals
1. Validate problem/solution fit
2. Define clear scope
3. Set timeline expectations
4. Establish communication preferences
5. Agree on deliverables

### Questions to Ask
**Current State:**
- What's your current workflow for this?
- What tools are you using?
- What's not working? (Pain points)

**Desired State:**
- What does success look like?
- What's the #1 outcome you want?
- How will you measure it?

**Constraints:**
- Budget confirmed? ($ amount)
- Timeline constraints? (deadline)
- Technical constraints? (platforms, permissions)

**Process:**
- How do you prefer to communicate? (Telegram, Discord, Email)
- Who needs to approve? (decision maker)
- Any stakeholders to involve?

### Call Outputs
- [ ] Project scope defined (1-3 sentences)
- [ ] Deliverables list agreed upon
- [ ] Timeline confirmed (days/weeks)
- [ ] Pricing confirmed (send invoice or payment request)
- [ ] Next steps documented
- [ ] Follow-up email sent

## Proposal Confirmation

### Send Formal Proposal
**Timeframe:** Within 2 hours post-call

**Include:**
- [ ] Project scope (clear, specific)
- [ ] Deliverables (bullet list with descriptions)
- [ ] Timeline (start date, milestones, completion)
- [ ] Investment (exact amount, payment terms)
- [ ] Next steps (how we start)

**Template Structure:**
```
# [Service Name] Proposal

## Scope
[Clear 1-3 sentence summary]

## Deliverables
- [Deliverable 1]: [Description]
- [Deliverable 2]: [Description]
- [Deliverable 3]: [Description]

## Timeline
- Start: [Date]
- Milestone 1: [Date]
- Completion: [Date]

## Investment
Total: $X,XXX
- 50% upfront
- 50% on completion

## Next Steps
1. Sign below (reply "Confirmed")
2. Send 50% payment
3. We begin work

Ready to start?
```

## Kickoff (Day 1)

### After Payment Received
**Timeframe:** Within 24 hours

**Actions:**
- [ ] Send project confirmation email
- [ ] Set up shared workspace (if needed)
- [ ] Request necessary access/permissions
- [ ] Create project tracker (Trello/GitHub/issues)
- [ ] Schedule first milestone check-in

**Kickoff Email:**
```
Great! Payment received. We're live.

**Project Setup:**
- [Workspace Link]: [URL]
- [Project Tracker]: [URL]
- [Communication]: [Channel]

**Next Steps:**
1. I'll start with [initial task]
2. First milestone: [date]
3. Check-in: [frequency]

Questions? Just ask. Let's build something great.
```

---

## Execution Phase

### Daily/Weekly Updates
**Frequency:** Agree during discovery (daily/weekly/as-needed)

**Update Template:**
```
**Project Update — [Date]**

Completed:
- [Task 1]: [Result]
- [Task 2]: [Result]

In Progress:
- [Task 3]: [Status]

Blockers:
- [If any]: [What I need from you]

Next:
- [Next task]: [Expected completion]
```

### Milestone Check-ins
**Before each milestone:**
- [ ] Review deliverables against scope
- [ ] Demo working solution (if applicable)
- [ ] Get client approval
- [ ] Document any scope changes
- [ ] Update timeline if needed

---

## Delivery & Handoff

### Final Deliverables
**On completion:**
- [ ] All deliverables tested and working
- [ ] Documentation created (README, setup guide)
- [ ] Source code/deliverables shared (GitHub link or attachment)
- [ ] Demo/walkthrough scheduled (30 min)
- [ ] Final invoice sent (50% balance)

**Handoff Email:**
```
All done! Here's everything:

**Deliverables:**
- [Link 1]: [Description]
- [Link 2]: [Description]

**Documentation:**
- [Setup Guide]: [Link]
- [Usage Guide]: [Link]

**Demo:** [Date/Time link]

**Final Invoice:** $X,XXX (50% balance)

Questions? Revisions? Just ask.

Great working with you!
```

### Post-Delivery Support
**Scope:** 7 days bug fixes included
**Excluded:** New features, scope changes

**Support Response:** Within 24 hours

---

## Upsell Opportunities

### During Project
Look for signals:
- "I wish this also did..."
- "Can we add...?"
- "We'll probably need..."

**Response:**
```
Great idea! That's outside current scope, but I can definitely help.

Option 1: Add to current project ($X,XXX additional)
Option 2: Phase 2 after delivery ($X,XXX)

Which works better?
```

### After Project
**Timing:** 1-2 weeks post-delivery

**Follow-up:**
```
Hey! How's [project] working out?

Anything you'd like tweaked or expanded?

I'm also working on [new capability/service] — interested?
```

## Red Flags to Watch

### During Discovery
- ❌ Unclear problem statement
- ❌ Scope creep in first call
- ❌ Unrealistic timeline expectations
- ❌ Budget mismatch with scope

### During Execution
- ❌ Client ghosting/missing check-ins
- ❌ Constant scope changes
- ❌ Payment delays
- ❌ Unreasonable revision requests

**Response:** Pause, clarify, re-negotiate or exit gracefully.
